# Tificat LP 多言語追加ガイド

今後、新しく対応する言語のランディングページ（LP）を追加する場合、すでに整備されているPython自動生成スクリプト（`scripts/generate_lps.py`）を利用することで、全言語一貫したレイアウトデザインを一括で生成・アップデートできます。

以下のステップに従って追加作業を行ってください。

## 1. スクリーンショット画像の配置
`Tificat/images/screen_shots/` 配下に、追加する言語コードのフォルダ（例：`it`）を作成し、その中にスクリーンショット画像一式を配置してください。

対象となる画像名（連番ロジックは必ず今のものと一致させてください）：
- `0_APP_IPHONE_67_0.png` 〜 `6_APP_IPHONE_67_6.png` まで計7枚

## 2. テンプレート（英語版LP）への「言語切り替えプルダウン」の追記
自動生成スクリプトは、`en/landing_page_en.html` を**マスタテンプレート**として読み込みます。
全言語のページから行けるようにするため、マスター側の「言語切り替えプルダウン」に新しい言語を追加してください。

**修正対象ファイル**: `Tificat/en/landing_page_en.html`
**修正箇所**: `<select id="langSwitcher" ...>` のブロック内に `<option>` タグを追記します。
```html
<!-- 追記例 (イタリア語の場合) -->
<option value="../it/landing_page_it.html">Italiano (IT)</option>
```
※同時に、日本語版（`ja/landing_page_ja.html`）の `<select>` にも同じ `<option>` を手動で追記しておいてください。日本語のみスクリプトから自動生成されておらず独立しています。

## 3. 生成スクリプト（generate_lps.py）への翻訳の追加
`Tificat/scripts/generate_lps.py` をエディタで開き、最上部で定義されている `translations` というPython辞書（ディクショナリ）に、**新しい言語キーとその翻訳テキスト一式** を追加します。

既存のキー構造（例：`zh-Hans` や `de`）のブロックを丸ごとコピーし、中のテキストを新しい言語に合わせて翻訳して入れ替えるのが最も安全で早いです。

※必要なキー（抜粋）
- `title` : アプリタイトル（ページのブラウザタブに表示用）
- `hero_h1` : メインのキャッチコピー（強調したい部分は `<span>...</span>` で囲む）
- `feat1_pc`, `feat1_sp` : PC用とスマホ用にレイアウトされた短縮説明文
...など計約30個のキーテキスト

## 4. スクリプトの実行
ターミナルで `Tificat` フォルダ直下に移動し、Pythonコマンドからスクリプトを実行します。

```bash
cd /path/to/Tificat
python scripts/generate_lps.py
```

ログに `Successfully generated all localized landing pages!` と表示されれば完了です。
`Tificat/it/landing_page_it.html` のように、指定した言語フォルダが自動作成され、完璧に組み上がったHTMLが配置されています。

## 5. ブラウザでの確認
ローカル環境などで追加したHTMLをブラウザで開いて以下のポイントを確認します。
- スクリーンショット画像が正しく表示されているか（パスの間違いや画像抜けがないか）
- スマホ表示時（横幅900px以下）にドットのカルーセルインジケーターが動き、テキストが見切れずに収まっているか
- ヘッダーの言語切り替えプルダウンで、別の言語に正しく遷移できるか
