- 技術的な学びの共有


### ファイル命名規則
- 個別記事: `YYYY-MM-DD-slug.html`
- 例: `2025-01-15-awawapon-launch.html`
- slugは記事タイトルから生成（英数字とハイフンのみ）

## 🎨 デザイン方針

### 1. 既存デザインとの統一
- **カラーテーマ**: 既存の `--primary-color: #4682B4` を継承
- **フォント**: 'Helvetica Neue', Arial, sans-serif
- **カードデザイン**: 既存の `.app-card` スタイルを参考
- **レスポンシブ**: Bootstrap 5のグリッドシステムを活用

### 2. ブログ一覧ページ (blog/index.html)
- **レイアウト**: 2カラム（記事カード + サイドバー）
- **記事カード要素**:
  - サムネイル画像（任意）
  - タイトル
  - 公開日（YYYY年MM月DD日形式）
  - カテゴリ/タグ（将来的に）
  - 抜粋（最初の150文字程度）
  - 「続きを読む」リンク
- **サイドバー**（将来的に）:
  - 自己紹介
  - カテゴリ一覧
  - 最新記事
  - アーカイブ

### 3. 個別記事ページ (blog/posts/YYYY-MM-DD-slug.html)
- **レイアウト**: 1カラム（中央寄せ、最大幅800px）
- **要素**:
  - タイトル
  - 公開日
  - カテゴリ/タグ
  - 本文（Markdown風のスタイリング）
  - 前後の記事へのナビゲーション
  - 関連記事（将来的に）
  - シェアボタン（将来的に）

### 4. index.htmlへの追加
- **ナビゲーションバー**: 「開発日記」リンクを追加
- **Aboutセクション**（新規追加）:
  - ヒーローセクションの下に配置
  - 自己紹介
  - 開発理念
  - 開発日記へのリンク

## 📝 記事の構成例

### 基本構成
```html
<article>
  <header>
    <h1>記事タイトル</h1>
    <div class="meta">
      <span class="date">2025年1月15日</span>
      <span class="category">開発日記</span>
    </div>
  </header>
  <div class="content">
    <!-- 本文 -->
  </div>
  <footer>
    <!-- 前後の記事へのナビゲーション -->
  </footer>
</article>
```

### 記事スタイル
- 見出し: h2, h3, h4 を使用
- 段落: 適切な行間とマージン
- リスト: ul, ol のスタイリング
- コードブロック: シンタックスハイライト（将来的に）
- 引用: blockquote のスタイリング
- 画像: レスポンシブ対応

## ⚠️ リスクと対策

### リスク1: コンテンツの継続的な更新が困難
**対策**:
- 月1回の更新を目標に設定
- テンプレートを用意して執筆負担を軽減
- 既存の `_private_docs` 内の記事を活用

### リスク2: デザインの統一感が崩れる
**対策**:
- 既存のCSS変数とスタイルを継承
- テンプレートを統一
- デザインガイドラインの作成

### リスク3: SEO効果が出ない
**対策**:
- 構造化データの適切な実装
- キーワードリサーチの実施
- 内部リンク構造の最適化
- 定期的なコンテンツ更新

### リスク4: 多言語対応の負担
**対策**:
- まずは日本語のみで開始
- 重要な記事のみ英語版を作成
- 翻訳は段階的に実施
