# ブログ拡張戦略：複数コンテンツタイプ対応

## 📋 概要
開発日記以外のコンテンツ（個人日記、技術記事、フリーランス関連など）も将来的に扱えるよう、拡張性のあるフォルダ構成とファイル構成を提案する。

---

## 🎯 設計方針

### 基本原則
1. **拡張性**: 新しいコンテンツタイプを追加しやすい
2. **管理しやすさ**: ファイル構造が直感的で分かりやすい
3. **SEO最適化**: URL構造が検索エンジンに優しい
4. **ユーザビリティ**: ナビゲーションがシンプルで分かりやすい
5. **後方互換性**: 既存の構造を壊さない

---

## 🗂️ 推奨フォルダ構成案

### 案1: 統合ブログアプローチ（推奨）⭐

```
/
├── index.html (既存)
├── blog/
│   ├── index.html (全記事一覧 + カテゴリフィルタ)
│   ├── posts/
│   │   ├── dev/ (開発日記)
│   │   │   ├── 2025-01-15-first-post.html
│   │   │   ├── 2025-01-20-app-update.html
│   │   │   └── ...
│   │   ├── personal/ (個人日記・雑記)
│   │   │   ├── 2025-02-01-daily-thoughts.html
│   │   │   └── ...
│   │   ├── tech/ (技術記事)
│   │   │   ├── 2025-02-10-flutter-tips.html
│   │   │   └── ...
│   │   ├── freelance/ (フリーランス関連)
│   │   │   ├── 2025-02-15-freelance-life.html
│   │   │   └── ...
│   │   └── misc/ (その他)
│   │       └── ...
│   └── assets/
│       └── images/
│           └── posts/
│               ├── dev/
│               ├── personal/
│               ├── tech/
│               └── ...
```

**メリット**:
- ✅ カテゴリごとにフォルダが分かれていて管理しやすい
- ✅ URL構造が明確: `/blog/posts/dev/2025-01-15-first-post.html`
- ✅ カテゴリごとの一覧ページを作りやすい
- ✅ 画像もカテゴリごとに整理できる
- ✅ 将来的にカテゴリを追加するのが簡単

**デメリット**:
- ⚠️ カテゴリが増えるとフォルダが増える
- ⚠️ 1つの記事が複数カテゴリに属する場合の扱いが難しい

---

### 案2: フラット構造 + メタデータアプローチ

```
/
├── index.html
├── blog/
│   ├── index.html (全記事一覧 + カテゴリフィルタ)
│   ├── posts/
│   │   ├── 2025-01-15-dev-first-post.html
│   │   ├── 2025-01-20-dev-app-update.html
│   │   ├── 2025-02-01-personal-daily-thoughts.html
│   │   ├── 2025-02-10-tech-flutter-tips.html
│   │   └── ...
│   └── assets/
│       └── images/
│           └── posts/
```

**ファイル命名規則**:
- `YYYY-MM-DD-category-slug.html`
- 例: `2025-01-15-dev-first-post.html`
- 例: `2025-02-01-personal-daily-thoughts.html`

**メリット**:
- ✅ 全ての記事が1つのフォルダで見える
- ✅ 日付順に並べやすい
- ✅ 複数カテゴリに対応しやすい（メタデータで管理）

**デメリット**:
- ⚠️ カテゴリごとの整理がしにくい
- ⚠️ ファイル名が長くなる可能性

---

### 案3: マルチブログアプローチ

```
/
├── index.html
├── blog/ (開発日記)
│   ├── index.html
│   └── posts/
│       └── ...
├── diary/ (個人日記)
│   ├── index.html
│   └── posts/
│       └── ...
├── tech/ (技術記事)
│   ├── index.html
│   └── posts/
│       └── ...
└── ...
```

**メリット**:
- ✅ 完全に独立したブログとして管理できる
- ✅ 各ブログごとに異なるデザインを適用可能

**デメリット**:
- ⚠️ ナビゲーションが複雑になる
- ⚠️ 共通のスタイルやコンポーネントの管理が大変
- ⚠️ SEO的には分散してしまう

---

## 🎯 推奨案：統合ブログアプローチ（案1）

### 理由
1. **拡張性**: 新しいカテゴリを追加するのが簡単（フォルダを追加するだけ）
2. **管理しやすさ**: カテゴリごとに整理されている
3. **SEO**: URL構造が明確で検索エンジンに優しい
4. **ユーザビリティ**: カテゴリごとの一覧ページを作りやすい

---


### 2. URL構造

```
/blog/                          → 全記事一覧
/blog/?category=dev             → 開発日記のみ
/blog/?category=personal        → 個人日記のみ
/blog/posts/dev/2025-01-15-first-post.html  → 個別記事
/blog/posts/personal/2025-02-01-daily-thoughts.html  → 個別記事
```

### 3. 記事メタデータ構造

各記事のHTMLファイル内にメタデータを埋め込む：

```html
<!-- 記事メタデータ -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "記事タイトル",
  "datePublished": "2025-01-15",
  "dateModified": "2025-01-15",
  "author": {
    "@type": "Person",
    "name": "takekapp1990"
  },
  "articleSection": "開発日記",
  "keywords": ["Flutter", "アプリ開発", "開発日記"]
}
</script>

<!-- カテゴリ情報（HTMLコメントとして） -->
<!-- category: dev -->
<!-- tags: flutter, app-development, diary -->
```

### 4. ブログ一覧ページの機能

#### blog/index.html
- **全記事表示**: デフォルトで全カテゴリの記事を時系列順に表示
- **カテゴリフィルタ**: カテゴリボタンでフィルタリング
- **検索機能**: 将来的に実装
- **ページネーション**: 記事数が増えたら実装

#### カテゴリ別一覧ページ（将来的に）
- `/blog/dev/` → 開発日記のみ
- `/blog/personal/` → 個人日記のみ
- 各カテゴリごとに専用ページを作成可能

## 💡 ベストプラクティス

### 1. カテゴリの追加基準
- 記事が10本以上になったらカテゴリを検討
- 明確なテーマがある場合のみカテゴリを分ける
- 最初は少ないカテゴリから始める

### 2. 記事の分類
- 1つの記事は基本的に1つのカテゴリに属する
- 複数のカテゴリに該当する場合は、メインカテゴリを選ぶ
- タグ機能で細かい分類を補完（将来的に）

### 3. URLの一貫性
- カテゴリ名は英語で統一（URLに使用）
- 表示名は日本語・英語両方に対応
- slugは記事タイトルから自動生成
